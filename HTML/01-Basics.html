On CSS

CSS Statements

Four main components include the following:

  1. Declaration: the statement as a whole with curly braces.
  2. Selector: the name of the part you want to syle.
  3. Property: properties to edit (e.g., width, height, background, etc.)
  4. Value: numbers, colors, urls - values that we assign.

Classes

A CSS Class is an identifier of an element which can be used multiple times on a page.

  • Classes start with . in the selector .anything-you-want-to-name-it
  • The same class name can be used more than once within the same file
  • The class keyword refers to the class name that you set in the html page in the specific tag.
  • In the CSS file, you can identify a class on the whole, or you can identify particluar parts to style (ex: p tags, h3, or li tags)

Ids

  • Start with a pound (#)
  • Uses: header or an about section
  • Should only be used once per page - they are unique! Important with Javascript later on
  • Use for one-off styling

Classes vs Ids

  • Ids use '#name' in css: classes use '.name' in css
  • You can't have two ids on an element: you can have as many classes on an element as you want

Margins go on the outside of an element

Font

In CSS, there are two types of font family names:

  • generic family: a group of font families with a specific look (ex: serif)
  • font family: a specific font family (ex: "Times New Roman" or Arial)

The font-family property should hold several font names as a fallback in case the browser doesn't support the first font you picked.
Always start with the font you want, and end with a generic family. Also, if the font family is more than one word, it must be in
quotation marks (ex: "Times New Roman").

Here is a list of the CSS font properties:

font:
sets all the font properties in one declaration
font-family:
specifies the font family for text
font-size:
specifies the font size of text
font-weight:
specifies the weight of a font
